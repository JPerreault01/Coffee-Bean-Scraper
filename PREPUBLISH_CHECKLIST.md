# Pre-Publish Checklist

> **Run this on every draft before setting it to Published.** If you're using Claude Code, ask it to run through this checklist against the page's content and ACF fields.

---

## A. Content & Voice

- [ ] **Verdict is specific and direct** — no hedge words ("may", "could", "tends to", "some may find"). One sentence, opinionated.
- [ ] **No first-person consumption claims in analytical mode** — no "I tried", "I brewed", "I tasted" unless `personal_mode` flag is set in ACF.
- [ ] **No crowd attribution** — no "buyers say", "customers report", "verified reviews say". If referencing external opinion, use "publicly available customer feedback" or "aggregated public review data" only.
- [ ] **No fabricated data** — all sensory scores, prices, and origins must match verifiable product specs. If unsure, leave the field blank rather than guess.
- [ ] **No filler phrases** — "in conclusion", "it's worth noting", "overall", "at the end of the day" are banned.
- [ ] **Tasting notes are specific** — "dark chocolate bitterness that fades clean" not "rich chocolate notes".
- [ ] **Who it's for / Who should skip it** are filled in and specific — "espresso drinkers who want low acidity" not "coffee lovers".

---

## B. Required Disclosures

- [ ] **Affiliate disclosure near the top** of any page with affiliate links (FTC requirement). The `.cbi-disclosure-inline` block must appear above the first affiliate link.
- [ ] **AI content disclosure** on About / Editorial Standards page is in place and not removed.
- [ ] **Methodology disclosure** on About page is in place: "Reviews are generated using structured product data…"

---

## C. On-Page SEO

- [ ] **RankMath meta title** set. Format: `[Product] Review — [Roaster] | Coffee Bean Index` for reviews; `Best [X] for [Y] (2025) | Coffee Bean Index` for roundups.
- [ ] **RankMath meta description** set. 140–160 characters. Contains the primary keyword and a specific claim or data point (price, rating, verdict).
- [ ] **Primary keyword appears in H1** — the page title should contain the product name or target phrase.
- [ ] **Heading hierarchy** is H1 → H2 → H3 (no skipped levels). On bean pages, section headings (Who it's for, etc.) are H2s.
- [ ] **One H1 per page** — only the bean name or page title. No second H1.
- [ ] **Internal link to ≥1 origin guide** (e.g., "Ethiopian coffee" → `/origin/ethiopia/` or an editorial guide page).
- [ ] **All images have alt text** — describe the product/roaster, not "coffee image".
- [ ] **URL slug is clean** — lowercase, hyphenated, no stop words if possible. Matches the product name closely.

---

## D. Schema

- [ ] **`Product` + `Review` + `AggregateRating` + `BreadcrumbList` schema present** on bean pages — auto-generated by `cbi_bean_schema()` when ACF fields are filled.
- [ ] **`FAQPage` schema present** on bean pages — auto-generated when `tasting_notes`, `whos_for`, or `whos_not_for` fields are set.
- [ ] **`Article` schema present** on guide / roundup / comparison pages — auto-generated by template.
- [ ] **`ItemList` schema present** on taxonomy archive pages and the all-beans archive — auto-generated.
- [ ] Validate with [Google Rich Results Test](https://search.google.com/test/rich-results) before flagging as done. (Run from VPS after deploy, or use Unlighthouse when wired up.)

---

## E. ACF Fields (bean pages only)

- [ ] `verdict` — filled (required, ≤160 chars)
- [ ] `rating` — filled (1–10, 0.5 increments)
- [ ] `tasting_notes` — ≥3 specific bullets (one per line)
- [ ] `whos_for` — filled
- [ ] `whos_not_for` — filled
- [ ] `price_analysis` — filled with current vs. 30-day average
- [ ] `weight_oz` — filled
- [ ] `price_per_oz` — filled (auto-calculates from scraper if `product_id` is set)
- [ ] `current_price` — filled (auto-updates via scraper)
- [ ] `amazon_affiliate_url` OR `roaster_url` — at least one present
- [ ] `product_id` — matches ID in `products.json` (required for price chart)
- [ ] `last_reviewed` — set to today's date
- [ ] All taxonomy terms set: origin, roast-level, process-method, brew-method, roaster, flavor-note (≥2)

---

## F. Technical

- [ ] **Featured image set** — 1200×630px recommended. Alt text filled.
- [ ] **No console errors** — open browser DevTools on the published URL and check the Console tab.
- [ ] **Price chart loads** — if `product_id` is set, verify the `[coffee_price_chart]` shortcode renders a chart (not an error).
- [ ] **Mobile render** — check the page at 375px width. Key things: buy box is not sticky at mobile (it should drop inline), bean grid is single-column, hero text wraps cleanly.
- [ ] **Affiliate links open in new tab** with `rel="nofollow sponsored noopener"` — verified by inspecting the Buy on Amazon / Buy Direct links in the source.

---

## G. For Roundups and Comparisons

- [ ] Each pick in a roundup links to the full individual review page.
- [ ] Comparison pages declare a clear winner in the verdict box (`.vs-verdict-box`).
- [ ] If a roundup has ≥5 picks, a comparison summary table (`.comparison-table`) is present near the top.

---

**Sign-off:** Once all boxes are checked, set to Published and run `deploy-theme` if content only lives in the repo. Price data and schema update automatically on the next scraper run (06:00 daily).
